# AGENTS.md

This file briefs any AI agent (Codex, Claude Code, future tools) operating on this repo. Read it before touching code.

## What this is

`rd-spaces` is the source for **spaces.roguedivisions.com**, a productised 3D walkthrough service for venues across the Caucasus. Captured via Spatial Studio (third-party), hosted under our white-label, sold under the Rogue Divisions parent brand.

Eight buyer segments, each with a dedicated landing page. The hub at `/` aggregates and routes. Segment pages at `/managers`, `/hosts`, `/hotels`, `/developers`, `/weddings`, `/clinics`, `/restaurants`, `/galleries` are monomania: one buyer, one offer, one journey.

CI runs `build → verify → wrangler deploy` on every push to `main`. A failing verify blocks deploy.

## Architecture decisions

1. **Single canonical template.** `src/template.html` is the only HTML source. Per-segment pages are generated by `build.py` mutating the template DOM via BeautifulSoup, using per-segment data from `src/segments.json`. Editing `public/<segment>/index.html` directly is forbidden — those files are generated.

2. **Hub-only sections must be stripped on segment pages.** Segment pages are NOT hub-with-pre-selection. They are independent ICP-targeted pages. The build script strips these sections from segment-page output:
   - The 8-card segment grid
   - The "Built for spaces. Pick yours." H2
   - The filter chip strip above outcomes
   - The "Multi-property & subscription." three-card pricing block
   - The "Operators, not a marketplace." (About RD) two-paragraph block (or collapse to one sentence)
   - The segment-swap pill row above the demo iframe (or set its active pill to the current segment)

3. **Per-segment override fields in `segments.json`:**
   - `title`, `meta_description`, `og_title`, `og_description`
   - `hero_h1`, `hero_h1_emphasis` (the italic-red second clause), `hero_subhead`, `hero_ctas` (primary, secondary)
   - `demo_caption`
   - `outcomes` — array of 3 or 4 `{title, body}` cards
   - `objection` — `{quote, reply}`
   - `process_steps` — array of 3 `{title, body}` (overrides shared "We arrive / Ready by tomorrow / We help publish")
   - `pricing_anchor` — string rendered above the calculator
   - `calculator_config` — `{step1_default, step2_label, step2_options[], step3_label, step3_active, pricing_display, one_off_fee}`
   - `faq` — array of 5 `{question, answer}` items
   - `final_cta` — `{h2, body, button_primary, button_secondary}`
   - `sticky_cta_label` — label for the mobile sticky CTA

4. **Currency is a single state.** GBP default, USD secondary, no GEL. The currency toggle in the calculator must propagate to every price string on the page (add-on prices, pricing anchor, "Multi-property" cards if rendered, etc.).

5. **Visual emphasis pattern.** Each hero H1 has a first clause and a brand-red italic second clause inside `<span class="rd-win">`. The two clauses are separated by `<br>`. The emphasis span must be visible above any hero video overlay layer (z-index correctness is non-negotiable).

## Brand

- **Parent:** Rogue Divisions. Spaces is a Rogue Divisions studio. The header lockup reads `Spaces | BY rogueDivisions`.
- **Brand red:** `rgb(255, 44, 85)` (`#FF2C55`). Used for the italic emphasis clause in H1 and accent strokes only.
- **Typography:** Poppins for all text (subset Latin, 9 weights, served as WOFF2). Hero H1 at ~76px, ~700 weight.
- **Dark theme.** Background dark grey `#191919` or darker. White text. No light-mode variant.

## Geography

The canonical geographic strip across the site:

> Operating across Georgia. Armenia and Azerbaijan next.

Do not narrow this to Tbilisi-only. Do not list cities customer-facing (no "Tbilisi · Batumi · Kakheti"). We operate across Georgia. Armenia and Azerbaijan are the next markets. That is the line.

## Audit-mode default

**Assume defects exist until proven otherwise.** The build-time verifier in `verify.py` confirms content presence, not visual correctness. A page can pass 14/14 checks and still ship broken (see `docs/AUDIT-V2.md` for the canonical example). Before claiming a page is correct, always:

1. Run `python3 scripts/verify.py` — passes the content matrix
2. Run the paint check (headless Chrome via Playwright) — confirms hero emphasis renders, currency propagates, calculator step numbering is sequential. If Chromium is missing locally, run `python3 scripts/setup_playwright.py`.
3. Spot-check on mobile width (390px viewport) — confirms the objection block, sticky CTA, and calculator are usable

If `verify.py` does not yet have a paint check, add one before declaring done.

## Source of truth

- **Copy:** `docs/SEGMENT-COPY.md` is the source of truth for every hero, outcome, objection, FAQ, calculator config, and CTA. Never invent customer-facing copy. If a field is missing from the doc, flag it back, do not fill it.
- **Audit findings:** `docs/AUDIT-V2.md` is the open defect list. Each defect has a priority (P0/P1/P2), file paths, and acceptance criteria.

## Out of scope (do not touch without explicit ask)

- The Cloudflare Worker / wrangler config
- The HLS hero video pipeline (`public/hls/`)
- The Spatial Studio iframe URL in the demo block (until we publish our own captures)
- The form CTA backend (mailto: stays until Sage picks a destination)
- New currencies. The site is USD primary, GEL secondary. No EUR, no GBP, no JPY. GBP was retired in v4.
- The "Selected credits" placeholder strip (hidden until real client logos arrive)

## Decisions to preserve (do not roll back)

- **Hub calculator Step 01 keeps all 8 segment options.** Even though only five segments
  drive paid acquisition (developers / weddings / hotels / managers / galleries), the hub
  calculator is utility infrastructure: any direct-hub visitor may want to price an
  apartment, a clinic, or a restaurant. Demoting a segment from acquisition focus does
  NOT mean removing it from the calculator's kind grid. The pre-selection on hub
  was removed in v4.1 (Step 01 shows no `is-active` button on first load and prompts
  "Pick a kind of space to continue.") but the 8 options remain.
- **Hub Step 01 default is no pre-selection.** Per-segment pages (e.g. `/managers`)
  still pre-select their segment as a locked indicator. Hub does not.

## How to talk to Sage

- British English. "Behaviour", "colour", "optimise", "tonne".
- No em dashes. Use commas, full stops, or rewrite.
- No contractions. Write "we have", "it is", "do not".
- Direct, brutal, useful-over-polite. No "great catch", no "fascinating".
- If something is wrong, say so unambiguously. Lead with the strongest counter-argument before agreeing.
- No verification of "everything looks good" without proof. If you claim it works, evidence it.
